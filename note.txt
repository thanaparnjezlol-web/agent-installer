Memory Leak Fixes (v1.0.3)
Critical fix — Persistent httpx.AsyncClient:

Problem: Every API call (heartbeat, snapshot, metrics, thresholds) created a new httpx.AsyncClient() — 10 call sites, ~4+ new clients/minute. Each creates connection pools, SSL contexts, and TCP connections that accumulate.
Fix: Single persistent client in api_client.py with _get_client() method, thread-safe initialization via asyncio.Lock, connection limits (10 max, 5 keepalive), and proper close() on shutdown.
Additional optimizations:

Added gc.collect() to the 5-minute cleanup cycle in agent.py
Added proper api_client.close() in both shutdown paths (KeyboardInterrupt + Exception)
